# 类型包括：
# feat: 新功能
# fix: 修复bug
# doc: 文档
# style: 格式化 ESLint 调整等（不影响代码运行的变动）
# perf: 性能优化
# build: 构建工具、配置等的修改
# refactor: 重构，不是新增功能，也不是修改bug的代码变动
# test: 测试

# 影响范围包括：
# global：影响全局
# module: 影响某个模块、功能

# 主题：commit目的的简短描述，不超过50个字符
# body: 对本次commit的详细描述
# footer: 记录关闭bug的ID信息、变动理由、备注等
